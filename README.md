Описание проекта
Данный проект представляет собой курсовую работу по дисциплине «Базы данных», выполненную студентом 2-го курса факультета информатики и робототехники Уфимского государственного авиационного технического университета (УГАТУ).

Направление подготовки: 09.03.04 – Программная инженерия

Тема работы: Разработка концептуально-логических моделей базы данных бизнес-процесса

Бизнес-процесс
Автоматизация приемной комиссии вуза, включающая три основные функции:

Прием заявлений – регистрация заявлений и документов от абитуриентов

Зачисление – зачисление абитуриентов по общему конкурсу

Апелляции – рассмотрение апелляций по приему документов и результатам зачисления

Содержание работы
1. Проектирование внешних иерархических моделей
Разработаны иерархические модели данных для каждой автоматизируемой функции:

Функция 1 «Прием заявлений» – содержит сущность «Направление» с вложенными агрегатами

Функция 2 «Зачисление» – содержит сущность «Категория абитуриентов» с агрегатом «Зачисленный абитуриент»

Функция 3 «Апелляции» – содержит сущность «Апеллянт» с агрегатом «Апелляция»

Для каждой модели определены ограничения атрибутов, кортежей, уникальности и операционные правила.

2. Проектирование нормализованных локальных ER-моделей
Исходные иерархические модели преобразованы в нормализованные ER-модели (3НФ):

Вычленены многозначные и однозначные «скрытые» сущности

Определены связи между сущностями

Сформулированы правила ссылочной целостности

3. Проектирование глобальной ER-модели
Локальные модели объединены в единую глобальную модель:

Выявлены и слиты эквивалентные сущности («Абитуриент», «Сотрудник», «Направление», «Факультет»)

Созданы обобщающие суперсущности:

«Сотрудник» – с категориями «Секретарь», «Руководитель», «Член_комиссии»

«Справочник» – с категориями «Вид_документа», «Категория_абитуриента», «Льгота», «Вид_претензии»

Устранены дублирующиеся атрибуты и связи

Атрибуты-ссылки заменены явными связями

4. Проектирование внутренней реляционной модели
Глобальная ER-модель преобразована в реляционную в двух вариантах:

На основе натуральных ключей (5-24131404.sql)
Использование составных ключей из предметной области

Таблицы: Справка+, Сотрудник%ПЕРС, Секретарь:СОТ, Абитуриент%ПЕРС, Документ_абитуриента, Направление_абитуриента и др.

Внешние ключи ссылаются на натуральные ключи

На основе суррогатных ключей (6-24131404.sql)
Введение автоинкрементных идентификаторов (ID)

Таблицы с аналогичной структурой, но с первичными ключами типа INT AUTO_INCREMENT

Оптимизированы для производительности и упрощения ссылочной целостности

5. Проектирование моделей отображения
Разработаны представления и хранимые процедуры для связывания внешних иерархических моделей с внутренней реляционной моделью (8-24131404.sql):

вирт_Направление_предварительно – представление для отображения направлений с факультетами

проц_Выбрано_абитуриентом_предварительно – хранимая процедура для получения абитуриентов по направлению

проц_Документ_абитуриента – хранимая процедура для получения документов абитуриента

вирт_Выбрано_абитуриентом%Направление_предварительно – представление с полной информацией о выборах направлений

вирт_Направление_предварительно_Кол-во – представление для подсчета количества направлений по факультетам

Зачисленный_абитуриент	Зачисленные абитуриенты

Апелляционная_комиссия	Комиссии по апелляциям
Заседание_комиссии	Заседания комиссий
Апелляция	Апелляции абитуриентов
